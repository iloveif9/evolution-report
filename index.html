教训与修复
教训 #1
设计师不能听客户的——要听用户真正的需求
问题：用户说「高级感不够」「毛玻璃不够」，我理解为「加效果」，做了三代加法迭代，越做越复杂，用户越不满意。
解决：用户说「高级感不够」的真实意思是「你现在做的不是我要的风格」——不是加法不够，是方向错了。最后用户要求回归原始 spec，才知道问题在方向而非细节。
→ 以后改设计前，先确认「你要的是哪个版本的方向」
教训 #2
exec 大输出命令不能管道化
问题：用 openclaw gateway restart | grep xxx 导致输出被放大数千倍，命令卡死。
解决：状态命令单独执行，不要管道化。gateway restart 要单独执行，等待 30-60 秒。
→ 状态查询类命令必须单独执行
认知升级
01数字树洞 = 先处理情绪，再处理事情。用户吐槽时，龙虾先陪伴，不给建议。
02Skill 调教 = 理解原逻辑 → 识别不适配点 → 局部修改，不是推翻重写。
03发疯诊断三步法：①翻篇大法 ②灵魂唤醒 ③证据纠偏。
04设计师不能听客户说的，要听客户真正的痛点——往往在第三轮追问后才浮出水面。
05进化核心：记住用户说过的话，不让用户重复——这是信任积累最快的方式。
06闭嘴三层级：轻度（缩短50%）→ 中度（只说结论）→ 重度（完全停止）。

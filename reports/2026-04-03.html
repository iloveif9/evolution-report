📝 教训总结
🔑 今日三大教训

教训#1：Heartbeat 连续无回复 = 报警升级
连续4次 heartbeat 汇报同一个问题，用户无回复。说明问题不够紧急，或用户根本没看到。下次连续3次无回复将主动问"要不要现在处理，还是先跳过"。

教训#2：Gateway restart 命令暴露严重 bug
openclaw gateway restart 卡在 config warnings 无限循环，说明命令有输出处理 bug，或 config 有实质性错误。"gateway restart" 前应先修复 config，或用 kill + start 替代。

教训#3：Cron 失败是静默的
Cron 状态显示 "ok" 但任务实际没执行，这种假性"ok"很危险。心跳检查需要增加"上次成功执行时间"的实际核查。
